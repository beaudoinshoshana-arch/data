# 演示脚本

## 1. 开场

本系统面向污水厂曝气和 PAC 加药两个关键控制环节，目标是在保证 COD、氨氮、总磷和总氮稳定达标的同时，降低曝气能耗和药剂消耗。

## 2. 数据页讲解

打开大屏后先看顶部 KPI：融合数据规模、决策样本数量、安全可行率、传统定值对照节能节药、P95 响应时间和预测误差。说明系统使用本地单厂数据、国内公开监测数据和 WQP 联网补充数据构建场景库。

## 3. 预测页讲解

切换 COD、NH3-N、TP、TN 四个指标，展示实测、预测和排放限值曲线。说明监督代理模型用于预测下一小时出水，为控制器提供可快速调用的环境模型。

## 4. 推荐页讲解

查看曝气和药耗动作对比曲线，说明蓝线/绿线是 RL 推荐，基线是当前或传统动作。当前推荐卡展示推荐动作、目标函数改善和中文解释。

## 5. 安全性讲解

展示 Reward 雷达图和鲁棒性热力图，解释出水风险、能耗、药耗、平滑性、违规项，以及噪声、设备波动、传感器平直段下的可行率和达标率。强调所有 RL 动作必须经过安全盾和目标函数仲裁，如果越界或不优则裁剪或回退到局部专家搜索。

## 6. 创新总结

本项目创新点包括：深度融合场景库、曝气/加药双智能体、约束安全盾、可解释动作建议和实时控制论文的动态特征落地。

## 7. 风险与未来工作

当前 RL 为离线训练和代理环境验证，后续需要接入分钟级高频数据、现场执行反馈和操作员接受度数据，以完成真实闭环优化。
