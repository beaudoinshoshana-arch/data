# 分步反思记录

## 1. 赛题与现有产物梳理

- 结果：确认赛题要求覆盖数据采集、智能体模型、实时决策、可视化大屏、报告和系统对接。
- 风险：项目容易退化成单纯预测模型，无法体现智能体和控制闭环。
- 改进：用“监督代理模型 + Safe-MARL + 约束盾 + 专家回退”组织系统。

## 2. 数据融合

- 结果：生成融合数据注册表和场景库，本地单厂与公开监测数据统一到长表规范。
- 风险：外部数据字段不一定包含控制变量，直接混训会引入域偏移。
- 改进：外部数据默认用于分布边界、鲁棒性和场景扩增，只有字段可靠时才进入监督训练。

## 3. Safe-MARL

- 结果：实现曝气 agent 与投药 agent 的轻量 PyTorch 策略，所有动作通过安全盾。
- 风险：离线 reward 使用代理响应系数，不能替代现场闭环试验。
- 改进：保留局部专家搜索作为安全对照，并在大屏中暴露可行率、回退率和 reward 分解。

## 4. 后端 API

- 结果：FastAPI 提供摘要、时序、推荐、单点推理、RL 推荐和报告摘要接口。
- 风险：大规模 CSV 首次读取有延迟。
- 改进：使用缓存读取；后续可切换到 DuckDB/SQLite 做大数据查询。

## 5. 前端大屏

- 结果：React + ECharts 大屏展示实时 KPI、预测曲线、动作对比、reward 雷达图和数据源状态。
- 风险：ECharts 首包较大，移动端性能需关注。
- 改进：已配置 manual chunks；后续可按视图懒加载图表。
